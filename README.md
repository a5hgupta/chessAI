# chessAI
This project is a fully functional Chess Engine built in Python to deepen my understanding of object-oriented programming, algorithms, and game logic. While the project began as a learning exercise, it quickly became a challenging problem-solving experience that taught me how complex a seemingly simple game can be.
The engine implements all the core rules of chess, including legal move generation, check and checkmate detection, stalemate, castling, en passant, and pawn promotion. Every move is validated to ensure it follows official chess rules, while the game state is continuously updated after each move.
One of the most interesting discoveries during development was that the pawn, despite being the simplest-looking piece, required the most complicated logic. Handling first moves, diagonal captures, en passant, and promotion made its implementation far more challenging than pieces like the knight, whose movement pattern is unique but comparatively straightforward to code.
The engine also features an AI opponent powered by the Negamax algorithm, allowing the computer to evaluate board positions and choose competitive moves instead of playing randomly. This introduced me to concepts such as recursive search, board evaluation, and decision-making algorithms.
Throughout development, I encountered several debugging challenges. One of the toughest bugs caused the rook to disappear during castling, requiring careful inspection of move generation, board updates, and state management before identifying the issue. Solving problems like these significantly improved my debugging skills and understanding of software architecture.
